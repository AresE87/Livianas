- También auto-envía después de 3 segundos si la pestaña ya está visible
- Fallback: formulario manual si no hay email en localStorage
- Endpoint: `POST /api/send-materiales`

## QUÉ NO FUNCIONÓ

### 1. Checkout Pro API → Error 403
- Endpoint: `POST https://api.mercadopago.com/checkout/preferences`
- Error: `PA_UNAUTHORIZED_RESULT_FROM_POLICIES` (403)
- El token `APP_USR-...` no tiene permisos para crear preferencias de Checkout Pro
- Se intentó con USD y UYU, sin `notification_url` → mismo error
- **Decisión**: usar payment links estáticos en vez de Checkout Pro
- **Para investigar**: puede necesitar activar Checkout Pro en el dashboard de MP, o el token necesita scopes adicionales

### 2. Email del comprador en MP payment links
- Los payment links de MP **no exponen el email del comprador** de forma confiable
- El usuario hizo un pago de prueba (10 pesos) y preguntó: "cómo me llega el material si nunca di mi correo?"
- **Solución implementada**: recoger email ANTES del pago en nuestra página

### 3. `define:vars` en Astro
- `<script define:vars={{ mpLink }}>` daba error `mpLink is not defined`
- **Solución**: pasar el link via `data-mp-link` attribute en el form y leerlo con `dataset.mpLink`

### Flujo completo
```
/materiales (email input) → localStorage → MP (nueva pestaña) + /gracias (pestaña actual)
                                                                    ↓
                                                        visibilitychange detecta retorno
                                                                    ↓

                                                                    ↓
                                                        Resend envía email con PDFs
```
